# 🔍 为什么南二巷201房只显示2个床

## 问题分析

### 根本原因

您的小程序显示的床位数量是从**云开发数据库**中读取的实际数据，而不是配置文件中的设定值。

### 数据流程

```
配置文件（NAN_TWO_ROOM_CONFIG）
  → 后端计算统计（totalBeds = 3 * 2 = 6）
  → 数据库实际记录（只有1号床和2号床）
  → 前端显示（只显示2个床）
```

### 详细说明

#### 1. 配置文件定义（3个床）

```typescript
// server/src/modules/beds/beds.service.ts
const NAN_TWO_ROOM_CONFIG: Record<number, RoomConfig[]> = {
  2: [
    { room: '201', bedCount: 3, bedPositions: [1, 2, 3] },  // ← 配置了3个床
    ...
  ]
}
```

#### 2. 数据库实际记录（只有2个床）

在云开发数据库中，`beds` 集合中只有：
- 201房1号床（上铺）
- 201房1号床（下铺）
- 201房2号床（上铺）
- 201房2号床（下铺）

**缺少**：
- 201房3号床（上铺）❌
- 201房3号床（下铺）❌

#### 3. 后端接口返回数据

前端调用 `/api/beds/nantwo/floor/2/beds` 接口时，后端返回数据库中的实际记录，只有4条（2个床 × 2个位置）。

## ✅ 解决方案

### 方法 1：使用测试页面添加（推荐，30 秒）

1. **打开小程序**
   - 编译项目或扫码预览

2. **进入测试页面**
   - 找到「云开发测试页面」

3. **添加3号床**
   - 在「添加床位测试（南二巷201房）」区域
   - 床位编号输入：`3`
   - 点击「添加3号床位（上下铺）」

4. **刷新页面**
   - 返回楼层页面
   - 下拉刷新
   - 现在应该能看到3个床了！

#### 步骤 1：打开云开发控制台

1. 选择您的环境

```json
  "updated_at": "2025-01-08T00:00:00.000Z"
```

#### 步骤 4：添加下铺

再次点击「添加记录」，输入以下 JSON：

```json
{
  "dormitory": "nantwo",
  "floor": "2",
  "room": "201",
  "bed_number": "3",
  "position": "lower",
```

## ⚠️ 常见问题

### Q：添加后还是只显示2个床怎么办？

**A**：
1. 检查数据库中是否有3号床的记录
2. 在小程序中下拉刷新
3. 清除小程序缓存后重新打开

### Q：为什么配置文件中定义了3个床，但数据库中只有2个？

**A**：
- 配置文件只是定义了"应该有"多少个床位
- 数据库中的记录才是"实际有"多少个床位
- 需要手动添加或初始化床位记录

### Q：如何一次性添加所有房间的床位？

**A**：
目前需要手动逐个添加，或者可以编写初始化脚本批量添加所有配置的床位。
