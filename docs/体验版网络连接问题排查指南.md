## ❌ 常见问题与解决方案

### 问题 1：request 合法域名未配置（最常见）

**症状：**
- 体验版打开后，所有网络请求都失败
- 控制台报错：`request:fail url not in domain list`

**原因：**
微信小程序强制要求所有网络请求的域名必须在后台配置，否则会拦截请求。

**解决方案：**

#### 步骤 1：登录微信小程序后台
```
https://mp.weixin.qq.com
```

#### 步骤 2：配置 request 合法域名
1. 点击「开发」→「开发管理」
2. 点击「开发设置」
3. 找到「服务器域名」→「request 合法域名」
4. 点击「修改」
5. 添加以下域名：
   ```
   https://b287065f-8284-40e8-9ff3-f0a05eb1a25d.dev.coze.site
   ```
6. 点击「保存并提交」

#### 步骤 3：等待生效
- 域名配置需要 **15 分钟** 才能生效
- 生效后重新打开体验版测试

---

### 问题 2：域名配置还未生效

**症状：**
- 已经配置了合法域名
- 但体验版仍然无法连接网络
- 在开发工具中可以正常使用

**原因：**
微信小程序域名配置需要 15 分钟才能生效。

**解决方案：**
1. 确认已正确配置域名
2. 等待 15 分钟
3. 重新打开体验版测试
4. 如果仍然不行，尝试删除体验版，重新上传

---

### 问题 3：开发工具中可以正常使用，但体验版不行

**症状：**
- 在微信开发者工具中可以正常使用
- 上传体验版后无法连接网络

**原因：**
开发工具中勾选了「不校验合法域名」，所以可以正常使用。但体验版仍然需要配置合法域名。

**解决方案：**

#### 临时测试（仅用于排查）
1. 在微信开发者工具中
2. 点击「详情」→「本地设置」
3. 勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」
4. 重新打开体验版测试

**注意：**
- 这个选项只在开发工具中有效
- 体验版和正式版仍然需要配置合法域名
- 如果勾选后可以正常使用，说明确实是 request 合法域名配置的问题

---

### 问题 4：域名无法访问

**症状：**
- 已配置合法域名
- 但体验版仍然无法连接网络
- 浏览器也无法访问域名

**原因：**
后端服务未启动或域名有问题。

**解决方案：**

#### 步骤 1：测试域名是否可访问
在浏览器中访问：
```
https://b287065f-8284-40e8-9ff3-f0a05eb1a25d.dev.coze.site/api/export/stats
```

**预期结果：**
```json
{
  "code": 200,
  "msg": "success",
  "data": { ... }
}
```

#### 步骤 2：检查服务状态
如果浏览器无法访问，说明服务有问题，需要联系管理员检查后端服务。

---

### 问题 5：HTTPS 证书问题

**症状：**
- 浏览器访问域名时提示证书错误
- 体验版无法连接网络
- 控制台报错：`request:fail ssl error`

**原因：**
域名使用了自签名证书或证书过期。

**解决方案：**

#### 方案 1：在开发工具中临时测试
1. 在微信开发者工具中
2. 点击「详情」→「本地设置」
3. 勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」

#### 方案 2：修复证书问题
需要联系管理员修复 HTTPS 证书。

## 🚨 常见错误代码

| 错误代码 | 错误信息 | 原因 | 解决方案 |
|---------|---------|------|---------|
| `request:fail url not in domain list` | 域名未在白名单中 | request 合法域名未配置 | 配置 request 合法域名 |
| `request:fail timeout` | 请求超时 | 网络超时或服务无响应 | 检查网络和服务状态 |
| `request:fail ssl error` | SSL 证书错误 | HTTPS 证书问题 | 在开发工具中关闭证书校验或修复证书 |
| `request:fail` | 请求失败 | 其他网络问题 | 检查网络连接和服务状态 |
