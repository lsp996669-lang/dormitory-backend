# 后端服务重新部署指南

## 问题诊断

当前后端域名 `https://server-tan-nu-43.vercel.app` **已无法访问**（连接超时）。

**原因：**
- Vercel 免费版服务可能因休眠或过期已失效
- 需要重新部署后端服务

---

## 解决方案

### 方案一：重新部署到 Vercel（推荐）

#### 步骤 1：准备 GitHub 仓库

1. 在 GitHub 创建一个新仓库（如果还没有）
2. 上传后端代码到 GitHub

```bash
# 在本地执行
cd /workspace/projects/server
git init
git add .
git commit -m "Initial commit: NestJS backend for dormitory management"

# 添加远程仓库（替换为你的 GitHub 仓库地址）
git remote add origin https://github.com/你的用户名/dormitory-backend.git
git branch -M main
git push -u origin main
```

#### 步骤 2：在 Vercel 部署

1. 访问 [https://vercel.com](https://vercel.com)
2. 登录或注册账号（推荐使用 GitHub 账号登录）
3. 点击「Add New...」→「Project」
4. 导入刚才创建的 GitHub 仓库
5. 配置项目：

**Framework Preset:** Other
**Root Directory:** `server`
**Build Command:** `npm run build`
**Output Directory:** `dist`
**Install Command:** `npm install`

#### 步骤 3：配置环境变量

在 Vercel 项目设置中添加以下环境变量：

```bash
DATABASE_URL=postgresql://postgres:你的密码@db.你的项目.supabase.co/postgres
SUPABASE_URL=https://你的项目.supabase.co
SUPABASE_ANON_KEY=你的anon_key
```

**获取 Supabase 配置信息：**
1. 访问 [https://supabase.com](https://supabase.com)
2. 登录你的项目
3. 进入「Settings」→「API」
4. 复制以下信息：
   - Project URL → `SUPABASE_URL`
   - anon public → `SUPABASE_ANON_KEY`
5. 进入「Settings」→「Database」
6. 复制 Connection String (PostgreSQL) → `DATABASE_URL`

#### 步骤 4：部署

点击「Deploy」按钮，等待部署完成（约 2-3 分钟）。

#### 步骤 5：获取新域名

部署成功后，Vercel 会显示类似：
```
https://dormitory-backend-xxx.vercel.app
```

复制这个域名，这就是你的新后端地址。

### 方案二：使用本地开发环境（临时方案）

如果你只需要临时测试，可以使用本地开发环境：

#### 步骤 1：启动本地后端服务

```bash
cd /workspace/projects
coze dev
```

#### 步骤 2：使用内网穿透

由于本地 `localhost:3000` 无法被外网访问，需要使用内网穿透工具。

**推荐工具：ngrok**

1. 下载安装 ngrok：[https://ngrok.com](https://ngrok.com)
2. 启动内网穿透：

```bash
ngrok http 3000
```

3. 获取公网域名，类似：`https://abc123.ngrok.io`

#### 步骤 3：重新构建小程序

```bash
cd /workspace/projects
PROJECT_DOMAIN=https://abc123.ngrok.io pnpm build:weapp
```

#### 步骤 4：在微信小程序后台配置域名

将 `https://abc123.ngrok.io` 添加到 request 合法域名。

**注意：**
- ngrok 免费版域名会变化，每次重启都要重新配置
- 仅适合临时测试，不适合生产环境

### 方案三：部署到腾讯云开发（国内方案）

如果你希望使用国内服务，可以考虑腾讯云开发：

1. 访问 [https://cloud.tencent.com/product/tcb](https://cloud.tencent.com/product/tcb)
2. 创建云开发环境
3. 部署 Node.js 云函数
4. 获取后端域名
5. 配置到微信小程序

## 后续配置（无论使用哪种方案）

### 1. 重新构建小程序

将新域名配置到小程序：

```bash
cd /workspace/projects
PROJECT_DOMAIN=https://你的新后端域名 pnpm build:weapp
```

### 2. 配置微信小程序域名

1. 登录微信小程序后台：https://mp.weixin.qq.com
2. 进入「开发管理」→「开发设置」
3. 在 **request 合法域名** 中添加新域名：
   ```
   https://你的新后端域名
   ```
4. 等待 15 分钟让配置生效

### 3. 验证后端服务

```bash
curl https://你的新后端域名/api/health
```

应该返回：
```json
{"code":200,"msg":"success"}
```

## 常见问题

### Q1: Vercel 部署失败怎么办？

检查：
1. GitHub 仓库是否正确
2. Build Command 是否正确：`npm run build`
3. 环境变量是否配置完整
4. Supabase 数据库是否可访问

### Q2: ngrok 域名被微信拦截怎么办？

ngrok 免费版域名可能被微信识别为不安全域名，建议：
- 使用付费版 ngrok（自定义域名）
- 或改用 Vercel 部署

### Q3: 如何知道新域名是否可用？

```bash
# 测试域名是否可访问
curl -I https://你的新后端域名/api/health

# 测试 API 是否正常
curl https://你的新后端域名/api/floor-stats
```
