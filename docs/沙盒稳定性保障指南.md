#### 方案 C：定期自动备份（推荐）⭐

**步骤：**

```bash
# 1. 创建备份脚本
cat > /workspace/projects/scripts/backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/workspace/projects/backups"
mkdir -p $BACKUP_DIR

# 创建时间戳
TIMESTAMP=$(date +%Y%m%d-%H%M%S)

# 创建备份文件
tar -czf $BACKUP_DIR/backup-$TIMESTAMP.tar.gz \
  src/ \
  package.json \
  pnpm-lock.yaml \
  tsconfig.json \
  taro.config.ts \
  vite.config.ts \
  .env.local

# 只保留最近 7 天的备份
find $BACKUP_DIR -name "backup-*.tar.gz" -mtime +7 -delete

# 输出备份信息
echo "Backup created: backup-$TIMESTAMP.tar.gz"
ls -lh $BACKUP_DIR/backup-$TIMESTAMP.tar.gz
EOF

# 2. 添加执行权限
chmod +x /workspace/projects/scripts/backup.sh

# 3. 手动执行备份
/workspace/projects/scripts/backup.sh

# 4. 设置定时任务（可选）
# 编辑 crontab
crontab -e

# 添加以下行（每天凌晨 2 点自动备份）
0 2 * * * /workspace/projects/scripts/backup.sh >> /workspace/projects/backups/backup.log 2>&1
```

**优点：**
- ✅ 自动备份
- ✅ 定期执行
- ✅ 自动清理旧备份

**缺点：**
- ⚠️ 需要配置定时任务

#### 自动重启脚本（推荐）⭐

**步骤：**

```bash
# 1. 创建监控脚本
cat > /workspace/projects/scripts/monitor.sh << 'EOF'
#!/bin/bash

LOG_FILE="/tmp/coze-logs/dev.log"
MAX_LOG_SIZE=$((50 * 1024 * 1024)) # 50MB

# 检查日志文件大小
if [ -f "$LOG_FILE" ] && [ $(stat -f%z "$LOG_FILE" 2>/dev/null || stat -c%s "$LOG_FILE") -gt $MAX_LOG_SIZE ]; then
  echo "[$(date)] Log file too large, rotating..." >> $LOG_FILE
  mv $LOG_FILE "${LOG_FILE}.old"
  touch $LOG_FILE
fi

# 检查服务是否运行
if ! pgrep -f "coze dev" > /dev/null; then
  echo "[$(date)] Coze dev server not running, restarting..." >> $LOG_FILE
  cd /workspace/projects && coze dev >> $LOG_FILE 2>&1 &
fi

# 检查服务是否响应
if ! curl -s http://localhost:3000/api/health > /dev/null; then
  echo "[$(date)] Service not responding, restarting..." >> $LOG_FILE
  pkill -f "coze dev"
  sleep 5
  cd /workspace/projects && coze dev >> $LOG_FILE 2>&1 &
fi
EOF

# 2. 添加执行权限
chmod +x /workspace/projects/scripts/monitor.sh

# 3. 设置定时任务（每 5 分钟检查一次）
crontab -e

# 添加以下行
*/5 * * * * /workspace/projects/scripts/monitor.sh >> /workspace/projects/backups/monitor.log 2>&1
```

**优点：**
- ✅ 自动监控服务状态
- ✅ 服务异常时自动重启
- ✅ 日志自动轮转
