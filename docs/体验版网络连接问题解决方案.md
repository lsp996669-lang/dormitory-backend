## 常见问题排查

### Q1: 配置后还是无法连接？

**检查清单：**

- [ ] 域名是否完全正确（逐个字符检查）
- [ ] 域名是否以 `https://` 开头
- [ ] 是否已等待 15 分钟
- [ ] 是否点击了「保存」按钮
- [ ] 是否配置在「request 合法域名」位置（不是 uploadFile 或 downloadFile）

**验证方法：**
在微信小程序后台的「开发设置」页面，找到「request 合法域名」，确认列表中是否有：
```
https://b287065f-8284-40e8-9ff3-f0a05eb1a25d.dev.coze.site
```

### Q2: 为什么开发工具可以，体验版不行？

**原因：**
- 开发工具可以开启「不校验合法域名」选项
- 体验版和正式版强制检查域名白名单
- 这是微信的安全机制，无法绕过

### Q3: 域名配置在错误的位置怎么办？

**检查位置：**
- ❌ uploadFile 合法域名（用于文件上传）
- ❌ downloadFile 合法域名（用于文件下载）
- ❌ socket 合法域名（用于 WebSocket）
- ✅ request 合法域名（用于普通 HTTP/HTTPS 请求）

**修正方法：**
如果配置在错误位置，需要：
1. 删除错误位置的域名
2. 在「request 合法域名」位置重新添加

### Q4: 域名配置后多久生效？

- 通常 5-15 分钟
- 最长可能 30 分钟
- 期间请勿反复修改

### Q5: 如何确认域名配置成功？

**成功标志：**
- 体验版可以加载楼层信息
- 可以进行入住、搬离操作
- 可以进行点名操作
- 可以导入导出数据

### Q6: 体验版报错 "request:fail url not in domain list"

**含义：**
- 请求的 URL 不在合法域名列表中
- 解决方法：检查域名配置是否正确

### Q7: 体验版报错 "request:fail"

**含义：**
- 网络请求失败
- 可能原因：
  1. 域名未配置
  2. 域名配置未生效
  3. 后端服务异常
  4. 网络问题

**排查步骤：**
1. 检查域名是否配置
2. 等待 15 分钟后再试
3. 在浏览器中访问后端域名，确认服务正常
4. 检查手机网络连接
