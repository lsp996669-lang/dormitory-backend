# 沙盒链接断开问题解决方案

## 🔍 问题描述

### 问题现象
在使用小程序的「数据导入」功能时，经常出现沙盒链接断开的问题：

```
Image: [https://code.coze.cn/api/sandbox/coze_coding/file/proxy?expire_time=-1&file_path=assets%2Fimage.png&nonce=...&project_id=...&sign=...]
```

### 问题原因
代码中使用了 **Coze 临时文件代理链接**，这种链接存在以下问题：

1. **临时性**：即使设置了 `expire_time=-1`（永不过期），仍然会因为以下原因失效：
   - 沙盒会话过期
   - 签名验证失败
   - Coze 服务重启
   - 网络连接中断

2. **不稳定性**：
   - 依赖沙盒环境
   - 沙盒重启后链接失效
   - 无法保证长期可用性

3. **硬编码问题**：
   - 链接直接写在代码中
   - 无法动态更新
   - 维护困难

## ✅ 解决方案

### 方案 1：移除临时链接（已采用）⭐

**实施步骤：**

1. **删除硬编码的 Coze 链接**
2. **删除 `handleImportFromUrl` 函数**
3. **删除「南四巷住宿名单」卡片**
4. **只保留「上传Excel文件」功能**

**修改后的代码：**

```typescript
// src/pages/import/index.tsx

import { View, Text } from '@tarojs/components'
import { useState } from 'react'
import Taro from '@tarojs/taro'
import { Card, CardContent } from '@/components/ui/card'
import { Button } from '@/components/ui/button'
import { Upload, CircleCheck, CircleX, CircleAlert } from 'lucide-react-taro'
import { Network } from '@/network'
import './index.css'

interface ImportResult {
  checkIns: number
  checkOuts: number
  errors: string[]
}

const ImportPage = () => {
  const [importing, setImporting] = useState(false)
  const [result, setResult] = useState<ImportResult | null>(null)

  // ✅ 只保留上传文件功能
  const handleUploadFile = async () => {
    try {
      // 选择文件
      const res = await Taro.chooseMessageFile({
        count: 1,
        type: 'file',
        extension: ['xlsx', 'xls']
      })

      if (res.tempFiles && res.tempFiles.length > 0) {
        const file = res.tempFiles[0]

        Taro.showLoading({ title: '正在导入...', mask: true })

        const uploadRes = await Network.uploadFile({
          url: '/api/import/file',
          filePath: file.path,
          name: 'file'
        })

        Taro.hideLoading()

        const data = JSON.parse(uploadRes.data)

        if (data.code === 200 && data.data) {
          setResult(data.data)

          if (data.data.checkIns > 0 || data.data.checkOuts > 0) {
            Taro.showToast({
              title: `入住${data.data.checkIns}人，搬离${data.data.checkOuts}人`,
              icon: 'success',
              duration: 3000
            })
          }
        } else {
          throw new Error(data.msg || '导入失败')
        }
      }
    } catch (error: any) {
      Taro.hideLoading()
      console.error('[Import] 导入失败:', error)
      Taro.showToast({
        title: error.message || '导入失败',
        icon: 'none',
        duration: 3000
      })
    } finally {
      setImporting(false)
    }
  }

  return (
    <View className="min-h-screen bg-gray-50 p-4">
      <View className="text-center mb-6">
        <Text className="text-xl font-bold text-gray-800 block">数据导入</Text>
        <Text className="text-sm text-gray-500 block mt-1">从Excel文件导入入住和搬离人员数据</Text>
      </View>

      {/* ✅ 只保留上传文件导入 */}
      <Card className="overflow-hidden">
        <CardContent className="p-4">
          <View className="flex items-center gap-3 mb-3">
            <View className="w-10 h-10 rounded-full bg-green-100 flex items-center justify-center">
              <Upload size={20} color="#22c55e" />
            </View>
            <View className="flex-1">
              <Text className="text-sm font-medium text-gray-800">上传Excel文件</Text>
              <Text className="text-xs text-gray-500">选择本地xlsx/xls文件导入</Text>
            </View>
          </View>
          <Button
            className="w-full bg-green-600 hover:bg-green-700 text-white"
            onClick={handleUploadFile}
            disabled={importing}
          >
            <View className="flex items-center justify-center gap-2">
              <Upload size={16} color="#fff" />
              <Text className="text-white">选择文件</Text>
            </View>
          </Button>
        </CardContent>
      </Card>

      {/* ... 导入结果和使用说明 */}
    </View>
  )
}

export default ImportPage
```

**优点：**
- ✅ 完全避免临时链接问题
- ✅ 用户可以上传自己的文件
- ✅ 更灵活，更稳定
- ✅ 代码更简洁

**缺点：**
- ⚠️ 用户需要准备 Excel 文件
- ⚠️ 没有预设示例文件

### 方案 2：上传到对象存储（推荐用于需要示例文件的场景）

**实施步骤：**

1. **将 Excel 文件上传到对象存储（TOS）**
2. **使用持久化的 URL**
3. **通过后端 API 获取文件 URL**

**示例代码：**

```typescript
// 后端：上传文件到 TOS
async uploadFileToTOS(file: File): Promise<string> {
  const tosClient = new TOS({
    accessKeyId: process.env.TOS_ACCESS_KEY_ID,
    accessKeySecret: process.env.TOS_ACCESS_KEY_SECRET,
    region: process.env.TOS_REGION,
    bucket: process.env.TOS_BUCKET
  })

  const key = `excel/${Date.now()}_${file.name}`
  await tosClient.putObject(key, file)

  // 返回持久化的 URL
  return `https://${process.env.TOS_BUCKET}.${process.env.TOS_REGION}.tos-cn-beijing.volces.com/${key}`
}

// 前端：从后端获取文件 URL
const handleImportFromUrl = async () => {
  try {
    // 从后端获取持久化的文件 URL
    const res = await Network.request({
      url: '/api/import/get-sample-file',
      method: 'GET'
    })

    const fileUrl = res.data.data.url

    // 使用持久化 URL 导入
    const importRes = await Network.request({
      url: '/api/import/url',
      method: 'POST',
      data: { url: fileUrl }
    })

    // ... 处理导入结果
  } catch (error) {
    console.error('[Import] 导入失败:', error)
  }
}
```

**优点：**
- ✅ URL 持久化，不会失效
- ✅ 可以随时更新文件
- ✅ 支持版本管理
- ✅ 可以提供多个示例文件

**缺点：**
- ⚠️ 需要配置对象存储
- ⚠️ 需要额外的后端开发

### 方案 3：提供文件下载功能（推荐用于教学和测试）

**实施步骤：**

1. **在后端提供文件下载接口**
2. **前端提供「下载示例文件」按钮**
3. **用户下载后，使用「上传Excel文件」功能导入**

**示例代码：**

```typescript
// 后端：提供文件下载接口
@Get('download-sample')
async downloadSampleFile(@Res() res: Response) {
  const fileStream = fs.createReadStream('assets/sample.xlsx')
  res.setHeader('Content-Type', 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet')
  res.setHeader('Content-Disposition', 'attachment; filename=sample.xlsx')
  fileStream.pipe(res)
}

// 前端：提供下载按钮

  <CardContent className="p-4">
    <View className="flex items-center gap-3 mb-3">
      <View className="w-10 h-10 rounded-full bg-blue-100 flex items-center justify-center">

      </View>
      <View className="flex-1">
        <Text className="text-sm font-medium text-gray-800">下载示例文件</Text>
        <Text className="text-xs text-gray-500">下载Excel模板文件</Text>
      </View>
    </View>
    <Button
      className="w-full bg-blue-600 hover:bg-blue-700 text-white"
      onClick={handleDownloadSample}
    >
      <View className="flex items-center justify-center gap-2">
        <Download size={16} color="#fff" />
        <Text className="text-white">下载文件</Text>
      </View>
    </Button>
  </CardContent>

// 下载示例文件
const handleDownloadSample = async () => {
  try {
    Taro.showLoading({ title: '正在下载...', mask: true })

    const res = await Taro.downloadFile({
      url: `${PROJECT_DOMAIN}/api/import/download-sample`
    })

    Taro.hideLoading()

    if (res.statusCode === 200) {
      Taro.showModal({
        title: '下载成功',
        content: '示例文件已下载，请使用「上传Excel文件」功能导入。',
        showCancel: false
      })
    }
  } catch (error) {
    Taro.hideLoading()
    Taro.showToast({
      title: '下载失败',
      icon: 'none'
    })
  }
}
```

**优点：**
- ✅ 用户可以下载示例文件
- ✅ 示例文件可以随时更新
- ✅ 用户可以参考示例文件格式

**缺点：**
- ⚠️ 需要用户下载后再上传
- ⚠️ 多了一个步骤

## ✅ 总结

### 问题根源
使用了 Coze 临时文件代理链接，这种链接不稳定，容易断开。

### 解决方案
移除临时链接，只保留「上传Excel文件」功能。

### 优点
- ✅ 完全避免临时链接问题
- ✅ 代码更简洁
- ✅ 更稳定可靠

### 建议
如果需要提供示例文件，建议使用方案 2 或方案 3：
- **方案 2（对象存储）**：需要长期稳定的示例文件 URL
- **方案 3（文件下载）**：教学和测试场景，用户需要参考示例格式
