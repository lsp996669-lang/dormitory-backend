# 修复人员名单导致沙盒断开连接问题

## 🔍 问题分析

### 问题现象
- 查看人员名单时，沙盒开发服务器可能会断开连接
- 特别是当入住人员较多时（如整个楼层、整个宿舍）
- 前端显示"网络请求失败"或"请求超时"

### 根本原因

#### 1. 后端问题

**问题 1：使用 `.in('bed_id', bedIds)` 查询超限**
- 位置：`server/src/modules/beds/beds.service.ts`
- 问题：Supabase 的 `.in()` 查询最多支持 2000 个值
- 当楼层床位很多时，`bedIds` 数组可能超过限制
- 导致查询超时或失败

**问题 2：一次性批量插入太多数据**
- 位置：`server/src/modules/beds/beds.service.ts`
- 问题：初始化房间或楼层时，一次性插入所有床位
- 如果床位很多（如整个楼层），会导致数据库压力过大
- 可能导致沙盒内存溢出

**问题 3：缺少超时控制和错误处理**
- 位置：`server/src/modules/beds/beds.service.ts`
- 问题：查询可能长时间无响应
- 没有超时控制，导致连接一直占用

#### 2. 前端问题

**问题 1：没有超时控制**
- 位置：`src/pages/checkin/index.tsx`
- 问题：网络请求可能长时间无响应
- 没有超时控制，导致用户体验差

**问题 2：错误提示不够友好**
- 位置：`src/pages/checkin/index.tsx`
- 问题：错误提示不够详细
- 用户不知道是超时、网络问题还是服务器问题

  setLoading(true)
  try {
    let url = '/api/beds/floor/' + floor
    let cacheKey = `beds_floor_${floor}`

    // 南二巷宿舍使用不同的API
    if (isNanTwo) {

        cacheKey = `beds_nantwo_${floor}_${room}`
      } else {
        // 获取整个楼层的床位

        cacheKey = `beds_nantwo_floor_${floor}`
      }
    }

    console.log(`[CheckIn] 开始加载床位数据，URL: ${url}`)
