# H5 登录功能测试说明

## 概述

本文档说明如何在 H5 开发环境下测试宿舍管理系统的登录功能。

## 系统架构

```
┌─────────────────┐       ┌─────────────────┐
│   H5 前端       │       │   后端服务      │
│  localhost:5000 │ ────► │ localhost:3000  │
│   (Taro + React)│       │  (NestJS)       │
└─────────────────┘       └─────────────────┘
        │                         │
        └───── Vite 代理 ─────────┘
         (/api -> localhost:3000)
```

## 测试步骤

### 1. 启动开发环境

```bash
cd /workspace/projects
coze dev
```

### 2. 访问 H5 前端

- **地址**: http://localhost:5000
- **登录页面**: http://localhost:5000/#/pages/login/index

### 3. 测试登录功能

#### 3.1 H5 开发环境模拟登录

在 H5 环境下，系统会自动使用模拟登录（无需真实的微信小程序 code）：

```typescript
// 前端代码会自动生成模拟 code
const code = `mock_code_${Date.now()}`
```

#### 3.2 登录测试步骤

1. 打开 http://localhost:5000/#/pages/login/index
2. 点击「管理员登录」按钮
3. 等待登录成功提示
4. 登录成功后，用户信息会存储在 localStorage

#### 3.3 后端日志验证

查看后端日志，确认登录请求：

```bash
tail -f /tmp/coze-logs/dev.log
```

应该看到类似日志：

```
登录请求: { code: 'mock_code_1234567890', nickname: '用户' }
检测到模拟登录 code，生成模拟 openid
获取到 openid: mock_openid_1234567890
创建新用户成功: { id: 'xxx', openid: 'xxx', is_approved: false, ... }
```

### 4. 邀请码验证

新用户登录后，`is_approved` 字段为 `false`，需要验证邀请码才能激活账户。

#### 4.1 默认邀请码

系统预设的测试邀请码：
- `TEST2024`
- `DORM2024`
- `ADMIN2024`

#### 4.2 验证邀请码（API 测试）

```bash
curl -X POST http://localhost:3000/api/auth/verify-invite \
  -H "Content-Type: application/json" \
  -d '{"userId":"用户ID","inviteCode":"TEST2024"}'
```

成功响应：

```json
{
  "code": 200,
  "msg": "验证成功",
  "data": {
    "id": "用户ID",
    "is_approved": true,
    ...
  }
}
```

### 5. 测试已验证用户登录

邀请码验证成功后，再次登录会更新用户信息，无需再次验证。

## 常见问题

### Q1: 登录按钮无响应

**可能原因**：
1. 后端服务未启动
2. 网络请求失败

**排查步骤**：
1. 检查后端服务状态：`curl http://localhost:3000/api/health`
2. 查看浏览器控制台错误
3. 检查后端日志：`tail -f /tmp/coze-logs/dev.log`

### Q2: 提示"登录失败"

**可能原因**：
1. 数据库连接失败
2. 网络请求超时

**排查步骤**：
1. 检查 Supabase 数据库连接
2. 查看后端日志中的错误信息

### Q3: 如何清除登录状态？

在浏览器控制台执行：

```javascript
localStorage.removeItem('userInfo')
location.reload()
```

## 后端接口说明

### 登录接口

- **URL**: `POST /api/auth/login`
- **参数**:
  ```json
  {
    "code": "微信登录 code 或 mock_code_xxx",
    "nickname": "用户昵称（可选）",
    "avatar": "用户头像 URL（可选）"
  }
  ```
- **响应**:
  ```json
  {
    "code": 200,
    "msg": "登录成功",
    "data": {
      "id": "用户ID",
      "openid": "用户 openid",
      "nickname": "昵称",
      "avatar": "头像",
      "is_host": false,
      "is_approved": false
    }
  }
  ```

### 邀请码验证接口

- **URL**: `POST /api/auth/verify-invite`
- **参数**:
  ```json
  {
    "userId": "用户ID",
    "inviteCode": "邀请码"
  }
  ```
- **响应**:
  ```json
  {
    "code": 200,
    "msg": "验证成功",
    "data": {
      "id": "用户ID",
      "is_approved": true,
      ...
    }
  }
  ```

## 开发环境配置

### Vite 代理配置

H5 开发环境通过 Vite 代理转发请求：

```typescript
// config/index.ts
h5: {
  devServer: {
    port: 5000,
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true,
      },
    },
  },
}
```

### 前端网络请求

前端使用 `Network.request` 发送请求：

```typescript
import { Network } from '@/network'

// 登录请求
const res = await Network.request({
  url: '/api/auth/login',
  method: 'POST',
  data: { code, nickname, avatar }
})
```

## 注意事项

1. **H5 开发环境**：使用模拟登录，无需真实的微信小程序 code
2. **生产环境**：需要配置真实的微信小程序 appid 和 secret
3. **邀请码**：生产环境应通过环境变量 `INVITE_CODES` 配置邀请码
4. **安全性**：模拟登录仅用于开发环境，生产环境必须使用真实的微信登录
