## 🚀 重新部署步骤

### 第一步：推送代码到 GitHub

```bash
cd /workspace/projects/server
git add .
git commit -m "fix: 支持微信云托管监听 0.0.0.0"
git push origin main
```

### 第二步：在微信云托管后台重新部署

1. **登录微信云托管**

   访问：[https://console.cloud.tencent.com/tcb](https://console.cloud.tencent.com/tcb)

2. **进入你的环境**

   - 选择环境：**prod-6gqi0mrc97ffd340**
   - 或点击你的项目名称

3. **进入服务管理**

   - 找到 **"云托管"** 或 **"服务"**
   - 点击 **"dormitory-api"** 服务

4. **配置环境变量**

   点击 **"环境变量"** 或 **"配置管理"**，添加：

   | 变量名 | 值 |
   |--------|-----|
   | HOST | `0.0.0.0` |
   | PORT | `3000` |
   | NODE_ENV | `production` |
   | COZE_SUPABASE_URL | 从微信云托管 PostgreSQL 获取 |
   | COZE_SUPABASE_ANON_KEY | 从微信云托管 PostgreSQL 获取 |

   **如何获取 PostgreSQL 配置：**
   - 在环境页面，找到 **"数据库"**
   - 点击你的 PostgreSQL 数据库
   - 查看 **"连接信息"**
   - 复制连接字符串和密钥

5. **重新部署**

   - 点击 **"部署"** 或 **"重新部署"**
   - 选择代码来源：**GitHub**
   - 选择仓库：**lsp996669-lang/dormitory-backend**
   - 选择分支：**main**
   - 构建目录：**server**
   - 构建命令：`npm run build`
   - 启动命令：`npm run start:prod`
   - 点击 **"部署"**

6. **等待部署完成**

   部署需要 2-3 分钟，完成后状态会显示为 **"运行中"**

## ❓ 常见问题

### Q1: 部署失败怎么办？

**检查：**
1. 构建命令是否为 `npm run build`
2. 启动命令是否为 `npm run start:prod`
3. 环境变量是否配置完整
4. PostgreSQL 连接信息是否正确

### Q2: 服务返回 404？

**检查：**
1. 确认已配置 HOST=0.0.0.0
2. 确认 PORT=3000
3. 确认路径包含 `/api` 前缀
4. 重新部署服务

### Q3: 数据库连接失败？

**检查：**
1. COZE_SUPABASE_URL 是否正确
2. COZE_SUPABASE_ANON_KEY 是否正确
3. PostgreSQL 数据库是否已创建
