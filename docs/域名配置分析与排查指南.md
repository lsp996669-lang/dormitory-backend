## 如果已配置但还是无法连接

### 检查 1：域名是否完全正确

逐个字符检查域名：

```
正确：https://b287065f-8284-40e8-9ff3-f0a05eb1a25d.dev.coze.site
错误：https://b287065f-8284-40e8-9ff3-f0a05eb1a25d.dev.coze.site/
错误：https://b287065f-8284-40e8-9ff3-f0a05eb1a25d.dev.coze.site:443
错误：http://b287065f-8284-40e8-9ff3-f0a05eb1a25d.dev.coze.site
错误：https://b287065f-8284-40e8-9ff3-f0a05eb1a25d.dev.coze.site/api
```

### 检查 2：配置是否已保存

- 在微信小程序后台的「开发设置」页面
- 确认域名列表中能看到 Coze 域名
- 如果看不到，需要重新添加并保存

- 域名配置后需要 **15 分钟** 生效
- 期间请勿反复修改
- 建议配置后等待 20 分钟再测试

### 检查 4：小程序代码是否使用了正确域名

小程序构建时使用的域名是：

```typescript
// 在 dist-weapp/common.js 中
PROJECT_DOMAIN = "https://b287065f-8284-40e8-9ff3-f0a05eb1a25d.dev.coze.site"
```

如果配置了其他域名，小程序仍然会使用 Coze 域名。

### 检查 5：后端服务是否正常

在浏览器或命令行中测试：

```bash
curl -I https://b287065f-8284-40e8-9ff3-f0a05eb1a25d.dev.coze.site/api/checkin/list
```

应该返回：
```

```

## 常见错误原因

### 错误 1：域名配置在错误位置

**症状**：配置了域名但还是无法连接

**原因**：配置到了 uploadFile 或 downloadFile，而不是 request

**解决**：
- 删除错误位置的配置
- 在「request 合法域名」中重新添加

### 错误 2：域名配置未生效

**症状**：配置了域名但立即测试，还是无法连接

**原因**：配置需要 15 分钟生效

**解决**：等待 15 分钟后再测试

### 错误 3：域名格式错误

**症状**：配置了域名但格式不正确

**原因**：域名带了端口号、路径或使用 http

**解决**：使用标准的 HTTPS 域名格式

### 错误 4：配置了失效的域名

**症状**：配置了多个域名，包含失效域名

**原因**：旧域名 `server-tan-nu-43.vercel.app` 已失效

**解决**：删除失效域名，只保留有效的 Coze 域名

### 错误 5：域名列表过长

**症状**：配置了大量域名，导致某些请求被拦截

**原因**：微信对域名数量有限制（request 最多 20 个）

**解决**：只保留必需的域名
