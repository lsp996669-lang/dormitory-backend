## 上传后操作

### 步骤 5：在微信小程序后台提交审核

1. 登录微信小程序后台：[https://mp.weixin.qq.com](https://mp.weixin.qq.com)
2. 进入「管理」→「版本管理」
3. 找到刚才上传的体验版
4. 点击「提交审核」
5. 填写审核信息：
   - **功能页面**：选择主要功能页面（如「宿舍管理」）
   - **测试账号**：（如有需要，可提供测试账号）
   - **用户标签**：（按需填写）
   - **审核说明**：
     ```
     宿舍管理助手 - 企业内部宿舍入住与搬离管理系统

     主要功能：
     - 宿舍楼层查看与床位管理
     - 入住登记与搬离管理
     - 点名功能（按楼层、宿舍）
     - 红名标记功能（重点关注人员）
     - 数据导入导出

     目标用户：企业内部宿舍管理人员

     备注：这是企业内部工具，不涉及商业行为
     ```

6. 点击「提交」

### 步骤 6：等待审核

- 审核时间：通常 1-3 个工作日
- 可在「版本管理」中查看审核进度
- 审核通过后，可以设置为「体验版」或正式发布

## 体验版使用

### 设置体验版成员

1. 在小程序后台，进入「设置」→「人员管理」→「体验者」
2. 添加体验者微信号（最多 20 个）
3. 体验者扫码后可体验小程序

### 体验版入口

- 体验者通过微信扫描体验版二维码
- 或通过小程序后台「版本管理」→「体验版」获取二维码

## 版本管理建议

### 版本号规范

使用语义化版本号：
- **主版本号**（Major）：重大功能更新或不兼容改动
- **次版本号**（Minor）：新增功能，向下兼容
- **修订号**（Patch）：bug 修复，向下兼容

示例：
- `1.0.0` - 首次发布
- `1.1.0` - 新增红名标记功能
- `1.1.1` - 修复点名宿舍过滤 bug
- `2.0.0` - 重大架构重构

### 版本备注建议

每次上传时填写清晰的版本备注，包括：
- 本次更新内容
- 修复的问题
- 新增的功能

示例：
```
v1.1.1 - 修复点名宿舍过滤问题

修复：
- 点名功能中南二巷和南四巷住户混在一起
- 页面标题显示宿舍信息

改进：
- 优化数据加载性能
```
