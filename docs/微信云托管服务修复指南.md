## 🚀 快速修复方案

### 方案 A：在微信云托管后台配置环境变量（推荐）

**操作步骤：**

1. **登录微信云托管**

   访问：[https://console.cloud.tencent.com/tcb](https://console.cloud.tencent.com/tcb)

2. **进入你的环境**

   - 环境ID：`prod-6gqi0mrc97ffd340`
   - 或点击你的项目名称

3. **进入服务管理**

   - 找到 **"云托管"** 或 **"服务"**
   - 点击 **"dormitory-api"** 服务

4. **配置环境变量**

   点击 **"环境变量"** 或 **"配置"**，添加：

   | 变量名 | 值 | 说明 |
   |--------|-----|------|
   | HOST | `0.0.0.0` | 监听所有网络接口 |
   | PORT | `3000` | 服务端口 |
   | NODE_ENV | `production` | 生产环境 |

5. **保存配置**

   点击 **"保存"** 或 **"确定"**

6. **重新部署服务**

   - 点击 **"重新部署"** 或 **"重启"**
   - 等待 1-2 分钟

## ❓ 如果服务还是返回 404

**可能原因：**

1. **路由没有正确映射**
   - 检查微信云托管的路由配置
   - 确认路径前缀是 `/api`

2. **服务没有正确启动**
   - 检查服务日志
   - 确认启动命令是 `npm run start:prod`

3. **环境变量没有生效**
   - 重新部署服务
   - 确认环境变量已保存

**解决方法：**

1. 查看服务日志
2. 检查启动日志中的错误
3. 根据错误信息修复
