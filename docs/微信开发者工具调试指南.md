# 微信开发者工具调试指南

## 问题说明

当在微信开发者工具中预览小程序时，显示"无法连接到服务器"，这是因为：

1. **云托管地址失效**：之前的配置指向云托管地址 `https://dormitory-api-xxx.sh.run.tcloudbase.com`，该服务可能已过期
2. **小程序网络限制**：小程序无法直接访问 `localhost`，需要特殊配置

---

## 解决方案

### 方案一：H5 开发调试（推荐，最简单）

直接在浏览器中访问 H5 版本，无需额外配置：

👉 **http://localhost:5000**

H5 版本通过 Vite 代理自动转发请求到本地后端（3000端口），所有功能正常。

---

### 方案二：微信开发者工具调试（需配置）

如果需要在微信开发者工具中调试小程序，请按以下步骤操作：

#### 第一步：重新构建小程序

已完成，构建产物在 `dist-weapp` 目录。

#### 第二步：配置微信开发者工具

1. 打开微信开发者工具
2. 导入项目，选择 `dist-weapp` 目录
3. **关键配置**：点击右上角「详情」→「本地设置」→ 勾选以下选项：
   - ✅ **不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**
   - ✅ **不校验安全域名、TLS 版本以及 HTTPS 证书**

![开发者工具配置示意](https://via.placeholder.com/400x200?text=勾选不校验域名选项)

4. 重新编译项目

#### 第三步：验证连接

刷新页面后，服务器状态应该显示为 🟢 在线。

---

### 方案三：使用内网穿透（小程序真机调试）

如果需要在真机上测试小程序，需要使用内网穿透：

#### 使用 ngrok

```bash
# 安装 ngrok
brew install ngrok  # macOS
# 或从 https://ngrok.com/download 下载

# 启动内网穿透
ngrok http 3000
```

记录生成的公网地址（如 `https://abc123.ngrok.io`），然后重新构建：

```bash
cd /workspace/projects
PROJECT_DOMAIN=https://abc123.ngrok.io pnpm build:weapp
```

#### 使用 localtunnel（备选）

```bash
# 安装
npm install -g localtunnel

# 启动
lt --port 3000
```

当前配置：

```env
PROJECT_DOMAIN=http://localhost:3000
```

- **H5 开发**：使用空字符串或 `http://localhost:3000`，Vite 代理会自动处理
- **小程序开发**：需要勾选「不校验合法域名」选项

## 常见问题

### Q1: 小程序仍然无法连接服务器？

**检查清单**：
1. 确认后端服务正在运行：`curl http://localhost:3000/api/health`
2. 确认微信开发者工具已勾选「不校验合法域名」
3. 点击开发者工具的「清除缓存」→「全部清除」，然后重新编译

### Q2: 如何查看网络请求日志？

在微信开发者工具中：
1. 打开「调试器」面板
2. 切换到「Network」标签
3. 查看请求状态和响应

### Q3: H5 和小程序有什么区别？

| 特性 | H5 | 小程序 |
|------|-----|--------|
| 访问方式 | 浏览器 | 微信 App |
| 网络请求 | Vite 代理 | 直接请求 |
| 本地调试 | 无需配置 | 需勾选不校验域名 |
| 真机测试 | 直接访问 | 需内网穿透 |

---

## 推荐开发流程

1. **日常开发**：使用 H5 版本（http://localhost:5000）
2. **小程序调试**：使用微信开发者工具 + 不校验域名选项
3. **真机测试**：使用内网穿透 + 小程序预览/体验版

---

## 服务状态检查

```bash
# 检查后端服务
curl http://localhost:3000/api/health

# 检查前端服务
curl http://localhost:5000

# 检查 API 代理
curl http://localhost:5000/api/export/stats
```

如果以上命令都返回正常，说明服务运行正常，问题在于小程序网络配置。
