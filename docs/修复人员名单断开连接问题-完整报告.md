# 修复人员名单断开连接问题 - 完整报告

## 📋 问题概述

### 问题描述
查看人员名单时，沙盒开发服务器可能会断开连接，导致：
- 请求失败
- 数据无法加载
- 用户体验差

### 影响范围
- 南四巷宿舍楼层查询
- 南二巷宿舍楼层查询
- 南二巷宿舍房间查询
- 初始化楼层床位

---

## 🔍 根本原因分析

### 1. 后端问题

#### 问题 1：`.in()` 查询超限
**位置：** `server/src/modules/beds/beds.service.ts`

**问题描述：**
- Supabase 的 `.in()` 查询最多支持 2000 个值
- 当楼层床位很多时，`bedIds` 数组可能超过限制
- 导致查询超时或失败

**影响：**
- 查询整个楼层人员时失败
- 查询南二巷整个楼层时失败

#### 问题 2：一次性批量插入太多数据
**位置：** `server/src/modules/beds/beds.service.ts`

**问题描述：**
- 初始化房间或楼层时，一次性插入所有床位
- 如果床位很多（如整个楼层），会导致数据库压力过大
- 可能导致沙盒内存溢出

**影响：**
- 初始化楼层床位时失败
- 沙盒内存溢出

#### 问题 3：缺少超时控制和错误处理
**位置：** `server/src/modules/beds/beds.service.ts`

**问题描述：**
- 查询可能长时间无响应
- 没有超时控制，导致连接一直占用

**影响：**
- 沙盒资源被占用
- 其他请求无法处理

### 2. 前端问题

#### 问题 1：没有超时控制
**位置：** `src/pages/checkin/index.tsx`

**问题描述：**
- 网络请求可能长时间无响应
- 没有超时控制，导致用户体验差

**影响：**
- 用户长时间等待
- 不知道是否需要刷新

#### 问题 2：错误提示不够友好
**位置：** `src/pages/checkin/index.tsx`

**问题描述：**
- 错误提示不够详细
- 用户不知道是超时、网络问题还是服务器问题

**影响：**
- 用户体验差
- 问题排查困难

#### 修复 1：分批查询入住记录

**文件：** `server/src/modules/beds/beds.service.ts`

**位置 1：获取房间床位（第 385-399 行）**

**修改前：**
```typescript

  .from('check_ins')
```

**修改后：**
```typescript
// 获取入住记录 - 使用关联查询避免 .in() 超限
let checkIns: any[] = [];
if (bedIds.length > 0) {
  // Supabase 的 .in() 查询最多支持 2000 个值，分批查询
  const batchSize = 500;
  for (let i = 0; i < bedIds.length; i += batchSize) {
    const batch = bedIds.slice(i, i + batchSize);
    const { data: batchCheckIns } = await client
      .from('check_ins')
      .select('id, bed_id, name, id_card, phone, check_in_time, is_station_marked, is_rider, station_name')
      .in('bed_id', batch);

    if (batchCheckIns) {
      checkIns.push(...batchCheckIns);
    }
  }
}
```

```typescript
const loadBeds = async () => {
  setLoading(true)
  try {
    let url = '/api/beds/floor/' + floor
    let cacheKey = `beds_floor_${floor}`

    // 南二巷宿舍使用不同的API
    if (isNanTwo) {

        cacheKey = `beds_nantwo_${floor}_${room}`
      } else {
        // 获取整个楼层的床位
```

```typescript
        cacheKey = `beds_nantwo_floor_${floor}`
      }
    }

    console.log(`[CheckIn] 开始加载床位数据，URL: ${url}`)

    // 添加超时控制（15秒）
```
