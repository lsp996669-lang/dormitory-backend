**根本原因：**
- 在 `beds.service.ts` 文件第 404 行，使用了未定义的变量 `bedIds`
- 这是在修复"人员名单断开连接问题"时引入的 bug
- 之前的修改将 `bedIds` 的定义移除了，但忘记在使用前定义它

---

## 🔧 修复过程

### 1. 定位问题代码

**文件：** `server/src/modules/beds/beds.service.ts`

**问题位置：** 第 398-404 行

**修复前的代码：**
```typescript
// 获取入住记录 - 使用关联查询避免 .in() 超限
let checkIns: any[] = [];
if (bedIds.length > 0) {  // ❌ bedIds 未定义
  // Supabase 的 .in() 查询最多支持 2000 个值，分批查询
  const batchSize = 500;
  for (let i = 0; i < bedIds.length; i += batchSize) {
    const batch = bedIds.slice(i, i + batchSize);
    // ...
  }
}
```

**问题：**
- `bedIds` 变量未定义
- TypeScript 编译失败
- 后端服务无法启动

---

### 2. 修复代码

**修复后的代码：**
```typescript
// 获取入住记录 - 使用关联查询避免 .in() 超限
let checkIns: any[] = [];
const bedIds = beds.map(b => b.id);  // ✅ 添加了 bedIds 的定义
if (bedIds.length > 0) {
  // Supabase 的 .in() 查询最多支持 2000 个值，分批查询
  const batchSize = 500;
  for (let i = 0; i < bedIds.length; i += batchSize) {
    const batch = bedIds.slice(i, i + batchSize);
    // ...
  }
}
```

**改进：**
- ✅ 在使用 `bedIds` 之前先定义它
- ✅ 从 `beds` 数组中提取所有床位 ID
- ✅ 确保变量正确初始化
