# 🎯 创建服务 - 详细配置指南

## 📋 根据当前界面填写信息

### 步骤1：填写基本信息

**服务名称：**
```
dormitory-api
```
⚠️ 注意：使用小写字母、数字和连字符，不要用中文

**服务介绍：**
```
宿舍管理后端服务
```
或
```
宿舍管理系统API服务
```

---

### 步骤2：选择部署方式

**✅ 正确选择：**
```
[●] 本地上传
[ ] 代码仓库
[ ] 镜像仓库
[ ] 模板创建
```

**选择"本地上传"**

---

### 步骤3：上传部署包

**点击"上传文件"按钮**

**选择文件：**
```
文件名: server-deploy.tar.gz
位置: /workspace/projects/server-deploy.tar.gz
大小: 26KB
```

**上传成功后会显示：**
```
✓ server-deploy.tar.gz (26KB)
```

---

### 步骤4：配置环境变量（关键步骤！）

**找到"环境变量"或"高级配置"区域**

**点击"添加环境变量"按钮，添加以下4个变量：**

#### 变量1：数据库URL
```
变量名: SUPABASE_URL
变量值: https://你的项目ID.supabase.co
```

#### 变量2：数据库密钥
```
变量名: SUPABASE_ANON_KEY
变量值: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```
⚠️ 注意：这个值很长，完整复制

#### 变量3：运行环境
```
变量名: NODE_ENV
变量值: production
```

#### 变量4：端口
```
变量名: PORT
变量值: 3000
```

---

### 步骤5：配置服务规格

**找到"规格配置"或"资源配置"区域**

**推荐配置：**
```
CPU: 0.5核 或 1核
内存: 512MB
最小实例: 1
最大实例: 2
```

**如果可选项较多，选择最基础的即可：**
- 最便宜的配置
- 或"按量付费"模式

---

### 步骤6：配置端口

**端口设置：**
```
服务端口: 3000
协议: HTTP
```

**如果需要配置健康检查：**
```
健康检查路径: /api/export/stats
健康检查端口: 3000
```

## ⚠️ 重要提醒

### 关于 Supabase 凭证

**如果您还没有 Supabase 凭证，需要先获取：**

**步骤：**
1. 登录 https://supabase.com
2. 进入您的项目
3. 点击左侧"设置" → "API"
4. 复制以下内容：
   - **Project URL** → 这就是 `SUPABASE_URL`
   - **anon public** → 这就是 `SUPABASE_ANON_KEY`

**示例：**
```
SUPABASE_URL=https://abcdefghijklmnop.supabase.co
SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImFiY2RlZmdoaWprbG1ub3AiLCJyb2xlIjoiYW5vbiIsImlhdCI6MTYxMjU0MzYwMCwiZXhwIjoxOTI4MTE5NjAwfQ.XXXXXXXXXXXXXXXXXXXX
```

## 💡 常见问题

### Q1: 找不到"环境变量"配置
**A:** 
- 点击"高级配置"或"更多设置"
- 或在"版本配置"页面配置

### Q2: 上传文件失败
**A:**
- 确认文件格式为 `.tar.gz`
- 确认文件大小不超过限制
- 尝试重新打包

### Q3: 服务名称格式错误
**A:**
- 只能使用小写字母、数字、连字符
- 不能使用中文、空格、特殊符号
- 例如：`dormitory-api` ✓

### Q4: 端口配置在哪？
**A:**
- 通常在"网络配置"或"高级配置"中
- 如果找不到，使用默认配置即可
