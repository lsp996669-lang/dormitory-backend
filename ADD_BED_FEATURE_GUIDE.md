# 🛏️ 添加床位功能使用指南

## 功能介绍

小程序新增了「添加床位」功能，允许管理员直接在小程序中为宿舍房间添加新床位（上下铺）。

---

## 📱 如何使用

### 方法 1：从楼层页面进入（推荐）

1. **登录小程序**
   - 确保已登录账号（只有登录用户才能看到添加床位按钮）

2. **进入楼层页面**
   - 在小程序首页，进入「楼层」页面
   - 在页面右上角，找到蓝色的「添加床位」按钮

3. **点击进入添加页面**
   - 点击「添加床位」按钮
   - 进入床位添加页面

---

### 方法 2：直接访问（开发调试）

1. 在微信开发者工具中
2. 在地址栏输入：`/pages/add-bed/index`
3. 或在 `app.config.ts` 中设置首页为该页面

---

## 🎯 添加床位的步骤

### 步骤 1：选择宿舍区域

点击「宿舍区域」下拉框，选择：
- **南四巷180号** - 南四巷宿舍
- **南二巷24号** - 南二巷宿舍

### 步骤 2：选择楼层

点击「楼层」下拉框，选择：
- **2楼** - 二层
- **3楼** - 三层
- **4楼** - 四层

### 步骤 3：选择房间号

点击「房间号」下拉框，选择具体的房间：
- 南四巷：1-15 号
- 南二巷：201、202、203、204、301、302、303、304、401、402、大厅

### 步骤 4：输入床位编号

在「床位编号」输入框中，输入要添加的床位编号：
- 例如：`3` 表示添加 3 号床
- 系统会自动创建上铺和下铺

### 步骤 5：点击添加按钮

点击「添加床位」按钮，系统会：
1. 调用后端 API 或云函数
2. 在数据库中创建上铺记录
3. 在数据库中创建下铺记录
4. 返回添加结果

## ⚠️ 注意事项

### 1. 权限要求

- **必须登录**：只有登录用户才能看到和使用添加床位功能
- 未登录时，楼层页面右上角不会显示「添加床位」按钮

### 2. 床位编号规则

- 只能输入数字
- 不允许重复添加相同编号的床位
- 例如：不能重复添加 201 房的 3 号床

### 3. 重复添加处理

如果尝试添加已存在的床位，系统会返回错误：

```
❌ 失败：该床位已存在：201房3号床
```

### 4. 支持的宿舍区域

#### 南四巷180号
- 楼层：2-4 楼
- 房间：每层 15 个床位
- 每个床位分为上铺和下铺

#### 南二巷24号
- 楼层：2-4 楼
- 房间配置：
  - 2 楼：201、202、203、204
  - 3 楼：301、302、303、304
  - 4 楼：401、402、大厅
- 每个床位分为上铺和下铺

## 🛠️ 技术实现

### 前端实现

**文件路径**：`src/pages/add-bed/index.tsx`

**核心功能**：
1. 宿舍区域选择器
2. 楼层选择器
3. 房间选择器（根据宿舍和楼层动态加载）
4. 床位编号输入
5. 调用云函数或后端 API

### 后端实现

**南二巷**：调用云函数 `addBed`
- 云函数路径：`cloudfunctions/addBed`
- 参数：`dormitory`, `floor`, `room`, `bedNumber`
- 返回：`code`, `msg`, `data`

**南四巷**：调用后端 API
- API 路径：`/api/beds/add`
- 方法：`POST`
- 参数：`dormitory`, `floor`, `room`, `bedNumber`
- 返回：`code`, `msg`, `data`

## 🆘 常见问题

### Q：为什么看不到「添加床位」按钮？

**A**：
1. 检查是否已登录
2. 未登录时，按钮不会显示
3. 请先登录后再试

### Q：添加床位失败，提示"云函数调用失败"

**A**：
1. 检查云开发环境是否已配置
2. 检查云函数是否已上传
3. 检查网络连接是否正常

### Q：添加成功后，在楼层页面看不到新床位

**A**：
1. 下拉刷新页面
2. 或返回上一页重新进入
3. 检查数据库中是否有记录

### Q：想删除已添加的床位怎么办？

**A**：
1. 在云开发控制台中打开 `beds` 集合
2. 找到对应的床位记录
3. 删除该记录（上铺和下铺都要删除）

### Q：如何批量添加多个床位？

**A**：
目前需要逐个添加，可以：
1. 重复添加操作
2. 或使用批量初始化脚本（`init-nantwo-beds.js`）

## 📊 使用场景

### 场景 1：新增房间

当新房间装修完成后，需要添加床位：
1. 选择对应的宿舍和楼层
2. 选择新房间号
3. 依次添加床位编号

### 场景 2：补充床位

当房间需要增加床位时：
1. 选择对应的宿舍、楼层、房间
2. 输入新的床位编号
3. 添加上铺和下铺

### 场景 3：修复数据

当数据库中缺少床位记录时：
1. 使用此功能补全缺失的床位
2. 确保数据库与实际情况一致
