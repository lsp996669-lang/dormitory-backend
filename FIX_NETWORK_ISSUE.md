# 🔧 修复小程序网络连接问题

## ❌ 问题描述

小程序无法连接网络，扫码预览也无法联网。

## 🎯 根本原因

1. **云开发环境未初始化**：项目使用云开发，但没有初始化云环境
2. **项目配置缺失**：`project.config.json` 中缺少云开发配置
3. **云函数未上传**：云函数代码未上传到云开发环境
4. **数据库集合未创建**：数据库集合未在云开发中创建

---

## 🛠️ 完整解决方案

### 第一步：配置云开发环境（必须）

#### 1. 打开微信开发者工具

1. 打开微信开发者工具
2. 导入您的项目（`/workspace/projects/`）
3. 确认 AppID 为：`wxeb1d51afc9237cda`

#### 2. 开通云开发

1. 点击工具栏上的「云开发」按钮
2. 点击「开通」（如果还没开通）
3. 选择「按量付费」或「免费试用」
4. 点击「开通」完成创建

#### 3. 获取云开发环境 ID

开通后，您会看到环境 ID，例如：`cloud1-7g1234567890`

**记录这个环境 ID，后面要用！**

---

### 第二步：更新代码配置

#### 1. 修改 app.tsx 中的环境 ID

打开 `/workspace/projects/src/app.tsx`，找到第 20 行：

```typescript
Taro.cloud.init({
  env: 'cloud1-7g1234567890', // ← 替换为您的实际环境 ID
  traceUser: true,
});
```

将 `cloud1-7g1234567890` 替换为您的实际环境 ID。

---

### 第三步：上传云函数

#### 1. 在微信开发者工具中上传云函数

1. 在微信开发者工具左侧，找到 `cloudfunctions` 文件夹
2. 依次右键点击以下文件夹，选择「上传并部署：云端安装依赖」：

   ```
   cloudfunctions/checkin
   cloudfunctions/checkout
   cloudfunctions/getCheckinList
   cloudfunctions/getCheckoutList
   cloudfunctions/exportData
   cloudfunctions/addBed
   ```

3. 等待每个云函数上传完成（显示绿色✓）

#### 2. 检查云函数是否上传成功

1. 打开云开发控制台：https://console.cloud.tencent.com/tcb
2. 选择您的环境
3. 点击「云函数」
4. 应该能看到 6 个云函数：
   - ✅ checkin
   - ✅ checkout
   - ✅ getCheckinList
   - ✅ getCheckoutList
   - ✅ exportData
   - ✅ addBed

---

### 第四步：创建数据库集合

#### 1. 在云开发控制台创建集合

1. 打开云开发控制台
2. 点击「数据库」
3. 点击「添加集合」
4. 依次创建以下 3 个集合：

   **集合 1：beds**
   - 集合名称：`beds`
   - 权限设置：选择「所有用户可读，仅创建者可写」

   **集合 2：checkin_records**
   - 集合名称：`checkin_records`
   - 权限设置：选择「所有用户可读，仅创建者可写」

   **集合 3：checkout_records**
   - 集合名称：`checkout_records`
   - 权限设置：选择「所有用户可读，仅创建者可写」

#### 2. 初始化床位数据（可选）

如果需要初始化床位数据，可以在 `beds` 集合中添加以下记录：

```json
{
  "location": "南四巷180号",
  "floor": "2楼",
  "beds": ["1", "2", "3", "4"]
}
```

---

### 第五步：测试网络连接

#### 1. 在微信开发者工具中测试

1. 编译项目（点击「编译」按钮）
2. 在模拟器中测试小程序
3. 进入「云开发测试页面」（`pages/test-cloud/index`）
4. 点击「测试入住登记」按钮
5. 查看是否成功

#### 2. 扫码真机预览测试

1. 点击「预览」按钮
2. 扫码在真机上测试
3. 进入「云开发测试页面」
4. 点击「测试入住登记」按钮
5. 查看是否成功

## 🔍 如何排查问题

### 问题 1：提示"云开发未初始化"

**原因**：环境 ID 配置错误或云开发未开通

**解决方案**：
1. 检查 `src/app.tsx` 中的环境 ID 是否正确
2. 确认云开发已开通
3. 重新编译项目

### 问题 2：提示"云函数调用失败"

**原因**：云函数未上传或上传失败

**解决方案**：
1. 检查云开发控制台，确认云函数已上传
2. 重新上传云函数
3. 查看云函数日志，排查错误

### 问题 3：提示"数据库操作失败"

**原因**：数据库集合未创建或权限配置错误

**解决方案**：
1. 检查数据库集合是否已创建
2. 检查集合权限设置
3. 确认集合名称拼写正确

### 问题 4：模拟器正常，真机无法连接

**原因**：真机网络问题或小程序权限问题

**解决方案**：
1. 检查手机网络连接
2. 确认小程序有网络权限
3. 重新扫码预览
