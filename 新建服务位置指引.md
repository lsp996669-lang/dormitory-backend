# 🔍 云托管控制台 - 新建服务位置指引

## 📍 新建服务的准确位置

### 情况一：已有云托管环境

**步骤1：进入云托管**
```
微信云开发控制台首页
↓
左侧菜单栏找到"云托管"
↓
点击"服务管理"
```

**步骤2：新建服务**
```
服务管理页面右上角
↓
"新建服务"按钮（蓝色/绿色按钮）
```

---

### 情况二：首次使用云托管

**步骤1：开通云托管**
```
左侧菜单 → 云托管
↓
如果是首次使用，会提示"开通服务"
↓
点击"立即开通"
```

**步骤2：创建环境（如需要）**
```
选择环境类型：
- 按量计费（推荐）
- 资源包

填写环境名称：
- 例如: dormitory-env
```

**步骤3：创建服务**
```
环境创建完成后
↓
进入"服务管理"
↓
点击"新建服务"
```

---

## 🔧 如果找不到"云托管"菜单

### 原因1：环境未开通云托管

**解决方案：**
```
1. 点击左侧"设置"
2. 找到"环境设置"
3. 开通"云托管"服务
```

### 原因2：使用的是旧版控制台

**解决方案：**
```
旧版路径：
云开发控制台 → 云应用 → 服务列表 → 新建服务

新版路径：
云开发控制台 → 云托管 → 服务管理 → 新建服务
```

### 原因3：权限不足

**解决方案：**
```
需要管理员权限才能创建服务
请联系小程序管理员授权
```

## 📋 新建服务填写内容

找到"新建服务"按钮后，点击并填写：

**基本信息：**
```
服务名称: dormitory-api
服务介绍: 宿舍管理后端服务
```

**部署方式：**
```
选择: 本地上传
```

**上传文件：**
```
选择: server-deploy.tar.gz (26KB)
```

**环境变量：**
```bash
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_ANON_KEY=你的密钥
NODE_ENV=production
PORT=3000
```

**规格配置：**
```
CPU: 0.5核
内存: 512MB
实例: 最小1，最大2
```

## 🎯 常见问题

### Q1: 左侧菜单没有"云托管"
**A:** 需要先开通云托管服务或升级环境

### Q2: 点击"云托管"后显示空白
**A:** 可能是环境问题，尝试刷新页面或更换浏览器

### Q3: 找不到"新建服务"按钮
**A:** 确认在"服务管理"页面，按钮通常在右上角

### Q4: 提示权限不足
**A:** 需要小程序管理员权限
