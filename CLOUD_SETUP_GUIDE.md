# 微信云开发配置指南

本指南将帮助你在微信开发者工具中配置云开发环境，使小程序能够使用云函数和云数据库。

## 前提条件

1. 已安装最新版本的微信开发者工具
2. 已登录微信开发者工具（使用小程序管理账号）
3. 已在小程序管理后台开通云开发服务

## 步骤 1：打开云开发控制台

1. 在微信开发者工具中打开项目
2. 点击顶部工具栏的「云开发」按钮
3. 如果是第一次使用，需要开通云开发服务：
   - 点击「开通」
   - 选择「按量付费」（适合测试环境）
   - 创建环境名称（如：dorm-management）
   - 选择基础版即可
   - 点击「确定」创建

## 步骤 2：配置环境 ID

1. 在云开发控制台中，点击「设置」->「环境设置」
2. 复制「环境 ID」（格式类似：dorm-management-xxxxx）
3. 在 `src/app.tsx` 文件中，找到云开发初始化代码：

```typescript
Cloud.init({
  env: 'your-env-id', // 替换为你的环境 ID
  traceUser: true
})
```

4. 将 `your-env-id` 替换为你的真实环境 ID

## 步骤 3：创建数据库集合

1. 在云开发控制台中，点击「数据库」
2. 创建以下 3 个集合：

### beds（床位表）
- **集合名称**：`beds`
- **权限设置**：所有用户可读，仅创建者可写
- **索引**：
  - `dormitory`（升序）
  - `floor`（升序）
  - `room`（升序）

### checkin_records（入住记录表）
- **集合名称**：`checkin_records`
- **权限设置**：所有用户可读，仅创建者可写
- **索引**：
  - `person_id`（升序）
  - `checkin_time`（降序）

### checkout_records（搬离记录表）
- **集合名称**：`checkout_records`
- **权限设置**：所有用户可读，仅创建者可写
- **索引**：
  - `person_id`（升序）
  - `checkout_time`（降序）

## 步骤 4：上传云函数

1. 在微信开发者工具的左侧目录树中，找到 `cloudfunctions` 目录
2. 依次右键点击以下 6 个云函数文件夹，选择「上传并部署：云端安装依赖」：

### 云函数列表

1. **checkin**（入住登记）
   - 功能：处理入住登记，生成二维码，添加床位信息

2. **checkout**（搬离登记）
   - 功能：处理搬离登记，更新床位状态，生成搬离记录

3. **getCheckinList**（获取入住列表）
   - 功能：获取当前所有入住人员列表

4. **getCheckoutList**（获取搬离列表）
   - 功能：获取历史搬离人员列表

5. **exportData**（导出数据）
   - 功能：导出入住数据为 Excel 文件

6. **addBed**（添加床位）
   - 功能：动态添加新床位（如南二巷 201 房的 3 号床）

3. 上传成功后，可以在「云开发」->「云函数」中看到所有云函数

## 步骤 5：配置云存储

1. 在云开发控制台中，点击「存储」
2. 创建以下目录结构：
   ```
   exports/
   ├── checkin/
   ├── checkout/
   └── temp/
   ```

3. 配置存储权限：
   - 创建目录后，点击「权限设置」
   - 选择「自定义安全规则」
   - 添加以下规则：

```json
{
  "read": true,
  "write": "auth != null"
}
```

## 步骤 6：测试云函数

1. 在微信开发者工具中，点击「云开发」->「云函数」
2. 点击任意云函数名称（如 `checkin`）
3. 点击「云端测试」
4. 输入测试参数：

### 测试 checkin 云函数
```json
{
  "dormitory": "南四巷180号",
  "floor": "2",
  "room": "201",
  "position": "upper",
  "bed_number": "1",
  "person_name": "张三",
  "person_id": "110101199001011234",
  "phone": "13800138000",
  "checkin_time": "2024-04-09 12:00:00",
  "payment_type": "公司支付",
  "payment_amount": 500,
  "remark": "测试入住"
}
```

5. 点击「测试」，查看返回结果：
   - 成功：返回 `{ code: 200, msg: "...", data: {...} }`
   - 失败：检查参数是否正确，云函数日志是否有错误

### 测试 addBed 云函数
```json

  "dormitory": "南二巷24号",
  "floor": "2",
  "room": "201",
  "bedNumber": "3"

```

## 步骤 7：验证数据库

1. 在云开发控制台中，点击「数据库」
2. 查看 `beds` 集合，应该能看到新增的床位记录
3. 查看 `checkin_records` 集合，应该能看到测试入住记录

## 常见问题

### Q1：云函数上传失败

**原因**：未开通云开发服务或环境 ID 配置错误

**解决**：
1. 确保已开通云开发服务
2. 检查 `src/app.tsx` 中的环境 ID 是否正确
3. 重新上传云函数

### Q2：数据库操作失败

**原因**：数据库权限设置不当

**解决**：
1. 检查集合权限设置
2. 确保创建索引
3. 检查云函数中的数据库操作代码

### Q3：云函数调用超时

**原因**：云函数执行时间超过限制

**解决**：
1. 检查云函数代码是否有死循环
2. 优化数据库查询语句
3. 在云函数配置中调整超时时间（最大 60 秒）

### Q4：获取云开发环境 ID 失败

**原因**：未正确初始化云开发环境

**解决**：
1. 确保 `src/app.tsx` 中正确调用了 `Cloud.init()`
2. 确保环境 ID 配置正确
3. 在微信开发者工具中测试云函数是否能正常调用

## 下一步

配置完成后，你可以：

1. 在小程序中测试「添加床位」功能
2. 在小程序中进行入住和搬离登记
3. 导出入住数据为 Excel 文件
4. 查看统计数据和红名人员列表

## 技术支持

如果遇到问题，请检查：

1. 云开发日志：在云开发控制台中查看云函数执行日志
2. 小程序控制台：在微信开发者工具的控制台中查看错误信息
3. 网络请求：在 Network 面板中查看请求是否成功
