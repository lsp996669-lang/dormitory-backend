# 云开发配置完整指南

## ✅ 第一步已完成：注册云开发

您已经成功注册了云开发环境！

## 🔧 接下来需要完成的步骤

### 步骤1：获取环境 ID（重要）

1. 在云开发控制台中，找到左上角的「环境 ID」
2. 复制环境 ID（格式类似：`cloud1-xxx` 或 `xxx-xxx`）
3. **将环境 ID 告诉我**，我会帮您确认配置

### 步骤2：在微信开发者工具中配置云开发

#### 2.1 打开项目
1. 打开微信开发者工具
2. 打开本项目（宿舍管理助手）

#### 2.2 配置云开发环境
1. 点击顶部菜单「云开发」
2. 点击「开通」（如果提示已开通则跳过）
3. 在弹出的窗口中选择刚创建的环境
4. 点击「确定」

#### 2.3 确认云开发已启用
1. 在左侧项目结构中，应该能看到 `cloudfunctions` 文件夹（云开发图标）
2. 右键点击 `cloudfunctions` 文件夹，应该能看到「当前环境：xxx」

### 步骤3：创建数据库集合

#### 3.1 打开云开发控制台
1. 点击顶部菜单「云开发」→「数据库」
2. 或者直接访问：https://console.cloud.tencent.com/tcb/databases

#### 3.2 创建 beds 集合
1. 点击「添加集合」
2. 集合名称输入：`beds`
3. 权限设置选择：**「所有用户可读，仅创建者可写」**
4. 点击「确定」

#### 3.3 创建 checkin_records 集合
1. 点击「添加集合」
2. 集合名称输入：`checkin_records`
3. 权限设置选择：**「所有用户可读，仅创建者可写」**
4. 点击「确定」

#### 3.4 创建 checkout_records 集合
1. 点击「添加集合」
2. 集合名称输入：`checkout_records`
3. 权限设置选择：**「所有用户可读，仅创建者可写」**
4. 点击「确定」

### 步骤4：上传云函数

#### 4.1 上传所有云函数
在微信开发者工具中，依次执行以下操作：

**上传 checkin 云函数：**
1. 右键点击 `cloudfunctions/checkin` 文件夹
2. 选择「上传并部署：云端安装依赖（不上传 node_modules）」
3. 等待上传完成（会显示绿色的对勾）

**上传 checkout 云函数：**
1. 右键点击 `cloudfunctions/checkout` 文件夹
2. 选择「上传并部署：云端安装依赖（不上传 node_modules）」
3. 等待上传完成

**上传 getCheckinList 云函数：**
1. 右键点击 `cloudfunctions/getCheckinList` 文件夹
2. 选择「上传并部署：云端安装依赖（不上传 node_modules）」
3. 等待上传完成

**上传 getCheckoutList 云函数：**
1. 右键点击 `cloudfunctions/getCheckoutList` 文件夹
2. 选择「上传并部署：云端安装依赖（不上传 node_modules）」
3. 等待上传完成

**上传 exportData 云函数：**
1. 右键点击 `cloudfunctions/exportData` 文件夹
2. 选择「上传并部署：云端安装依赖（不上传 node_modules）」
3. 等待上传完成

**上传 addBed 云函数（用于添加床位）：**
1. 右键点击 `cloudfunctions/addBed` 文件夹
2. 选择「上传并部署：云端安装依赖（不上传 node_modules）」
3. 等待上传完成

#### 4.2 验证云函数上传成功
1. 在云开发控制台中，进入「云函数」页面
2. 应该能看到 6 个云函数：checkin, checkout, getCheckinList, getCheckoutList, exportData, addBed
3. 每个云函数旁边应该显示绿色的状态图标

### 步骤5：配置云存储（用于导出 Excel）

#### 5.1 创建存储目录
1. 在云开发控制台中，进入「存储」页面
2. 点击「新建文件夹」
3. 文件夹名称输入：`exports`
4. 点击「确定」

#### 5.2 设置存储权限
1. 在「存储」页面，点击右上角的「权限设置」
2. 选择：**「所有用户可读，仅创建者可写」**
3. 点击「确定」

### 步骤6：测试云函数

#### 6.1 编译项目
1. 在微信开发者工具中，点击「编译」按钮
2. 等待编译完成

#### 6.2 访问测试页面
1. 在小程序预览中，点击「进入小程序」
2. 手动输入或跳转到测试页面：`pages/test-cloud/index`
3. **或者**：在地址栏输入：`/pages/test-cloud/index` 并回车

#### 6.3 测试各个云函数

**测试入住登记：**
1. 在「入住登记测试」部分，输入姓名和手机号
2. 点击「测试入住登记」
3. 应该提示「测试成功」

**测试获取入住列表：**
1. 在「获取入住列表测试」部分，点击「测试获取入住列表」
2. 应该提示「获取到 0 条记录」（因为没有数据）
3. 在云开发控制台的 `checkin_records` 集合中，应该能看到刚才创建的记录

**测试导出数据：**
1. 在「导出数据测试」部分，点击「测试导出数据」
2. 应该提示「导出成功」
3. 在云开发控制台的「存储」页面，`exports` 文件夹中应该能看到导出的 Excel 文件

**测试添加床位：**
1. 在「添加床位测试（南二巷201房）」部分，确认床位编号为 `3`
2. 点击「添加3号床位（上下铺）」按钮
3. 应该提示「添加成功」
4. 在云开发控制台的 `beds` 集合中，应该能看到新增的两条记录

## ❓ 常见问题

### Q1: 上传云函数时提示"上传失败"
**解决方案**：
- 检查网络连接
- 确认云开发环境已开通
- 尝试删除云函数后重新上传

### Q2: 云函数上传成功但调用失败
**解决方案**：
- 检查云函数代码是否有语法错误
- 在云开发控制台查看云函数日志
- 确认云开发环境 ID 配置正确

### Q3: 数据库操作提示"Permission denied"
**解决方案**：
- 检查数据库集合权限设置
- 确认权限为「所有用户可读，仅创建者可写」

### Q4: 找不到测试页面
**解决方案**：
- 在 app.config.ts 中检查 pages 数组是否包含 `pages/test-cloud/index`
- 在开发者工具中手动输入地址跳转
