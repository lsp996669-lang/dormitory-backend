
**打包上传：**
```bash

  server/src
```

#### 步骤2：登录云托管控制台

1. **访问控制台**
   ```
   https://cloud.weixin.qq.com
   ```

2. **选择环境**
   - 选择当前小程序对应的云开发环境
   - AppID: `wxeb1d51afc9237cda`

3. **进入云托管**
   ```
   左侧菜单: 云托管 → 服务管理
   ```

---

#### 步骤3：创建/更新服务

**如果已有服务：**
1. 找到服务 `dormitory-api`
2. 点击"新建版本"
3. 选择"本地上传"
4. 上传 `server-deploy.tar.gz`

**如果是新建服务：**
1. 点击"新建服务"
2. 选择"本地上传"
3. 填写服务信息：
   ```
   服务名称: dormitory-api
   服务介绍: 宿舍管理后端服务
   ```
4. 上传部署包

---

#### 步骤4：配置环境变量

**必须配置的环境变量：**

在"版本配置" → "环境变量"中添加：

```bash
# Supabase 数据库配置（必须）
SUPABASE_URL=你的Supabase项目URL
SUPABASE_ANON_KEY=你的Supabase匿名密钥

# 其他配置
NODE_ENV=production
PORT=3000
```

**获取 Supabase 凭证：**
1. 登录 [Supabase](https://supabase.com)
2. 进入项目设置 → API
3. 复制 `URL` 和 `anon public` 密钥

---

#### 步骤5：配置规格

**推荐配置：**
```
CPU: 0.5核
内存: 512MB
实例数量: 最小1个，最大2个
```

**高级配置：**
```
端口: 3000
协议: HTTP
健康检查: /api/export/stats
```

---

#### 步骤6：部署并验证

1. **点击"部署"**
   - 等待构建完成（约3-5分钟）
   - 查看构建日志

2. **验证部署**
   ```bash
   # 获取服务URL
   curl https://你的服务域名/api/export/stats
   
   # 测试搜索接口
   curl https://你的服务域名/api/checkin/search?keyword=138
   ```

3. **检查日志**
   - 在"版本管理" → "日志"中查看运行日志
   - 确认无错误信息

### 方式二：使用 Docker 本地构建后上传

#### 步骤1：构建 Docker 镜像

```bash
cd /workspace/projects/server

# 构建镜像
docker build -t dormitory-api:latest .

# 测试运行
docker run -d \
  -p 3000:3000 \
  -e SUPABASE_URL=你的URL \
  -e SUPABASE_ANON_KEY=你的KEY \
  dormitory-api:latest
```

#### 步骤2：推送到镜像仓库

**推送到腾讯云镜像仓库：**
```bash
# 登录
docker login ccr.ccs.tencentyun.com

# 标记镜像
docker tag dormitory-api:latest \
  ccr.ccs.tencentyun.com/你的命名空间/dormitory-api:latest

# 推送
docker push ccr.ccs.tencentyun.com/你的命名空间/dormitory-api:latest
```

#### 步骤3：在云托管中使用镜像

1. 新建版本时选择"镜像仓库"
2. 选择已推送的镜像
3. 配置环境变量
4. 部署

## ⚙️ 环境变量配置详解

### 必需环境变量

| 变量名 | 说明 | 获取方式 |
|--------|------|----------|
| `SUPABASE_URL` | Supabase 项目 URL | Supabase 控制台 → 项目设置 → API |
| `SUPABASE_ANON_KEY` | Supabase 匿名密钥 | Supabase 控制台 → 项目设置 → API |

### 可选环境变量

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `NODE_ENV` | production | 运行环境 |
| `PORT` | 3000 | 服务端口 |

## 🔍 常见问题

### 问题1：构建失败

**原因：** 依赖安装失败或代码编译错误

**解决：**
1. 查看构建日志
2. 检查 `package.json` 依赖版本
3. 确认 `pnpm-lock.yaml` 文件存在

### 问题2：服务启动失败

**原因：** 环境变量缺失或端口配置错误

**解决：**
1. 检查环境变量配置
2. 确认端口为 3000
3. 查看运行日志

### 问题3：接口返回 404

**原因：** 路由未正确配置

**解决：**
1. 确认代码中包含 `/api/checkin/search` 路由
2. 检查 Controller 装饰器
3. 重启服务

### 问题4：数据库连接失败

**原因：** Supabase 凭证错误或网络问题

**解决：**
1. 验证 SUPABASE_URL 和 SUPABASE_ANON_KEY
2. 检查 Supabase 项目状态
3. 确认网络连接正常
